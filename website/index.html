Workflow

How SupaPhone works

The system is pairing-first and backend-routed. Each browser and Android install gets a stable local identity, then the backend handles secure pairing, delivery, acknowledgements, and recent activity.

1. Pair the browser and phone

Open the extension and the Android app, then pair with the local QR code or the 6-digit pairing code.

2. Send from the browser

Use the context menu or quick-send input to route a link or number to one specific paired phone.

3. Act on Android

Android receives a notification and opens a chooser so the user can call, open the dialer, open a link, share, copy, or continue with WhatsApp based on the payload type.

Security and Privacy

Built to keep routing fast and controlled

SupaPhone uses backend-verified client secrets, short-lived pairing codes, minimal retention windows, and request throttling to protect onboarding and delivery flows.

Pairing safety

Pairing codes are short-lived, one-time consumed, and protected with backend throttling against repeated abuse.

Minimal permissions

Android requests notifications and call access with an in-app explanation, while camera is requested only when QR pairing is chosen.

Local QR generation

The browser extension now generates its pairing QR locally instead of sending pairing data to an external QR rendering service.
