# SupaPhone - Application Overview

## 1. Introduction

Application name: SupaPhone

Core concept:
SupaPhone is a productivity bridge between a desktop browser and an Android device. It lets a user send a phone number or web link from desktop to a paired phone through the browser extension, then finish the action on mobile with an explicit chooser.

## 2. Pairing

SupaPhone uses a pairing-first model with no login system.

Primary method:
- enter the 6-digit code generated by the extension

Alternate method:
- scan the QR shown in the extension

Connection model:
- multiple browsers can pair to one phone
- one browser can pair to multiple phones

## 3. Phone Payload Flow

### Desktop trigger

- right-click a phone number or selected phone-like text
- choose the paired device from the browser context menu

### Mobile flow

- Android receives the notification
- tapping the notification body opens the in-app chooser
- chooser options:
  - `Call`
  - `Open in dialer`
  - `WhatsApp`
  - `Share`

### Number handling

- Android applies region-aware normalization for WhatsApp routing
- direct call and dialer flows remain permission-aware

## 4. Link Payload Flow

### Desktop trigger

- right-click a hyperlink or page background
- choose the paired device from the browser context menu

### Mobile flow

- Android receives the notification
- tapping the notification body opens the in-app chooser
- chooser options:
  - `Open link`
  - `Copy link`
  - `Share link`

## 5. Ads Model

SupaPhone uses a constrained ad model:

- App Open Ad:
  - normal launcher-driven app open only
  - shown only during real startup work
  - never used to block notification-driven execution
- Inline banner ads:
  - allowed in supported in-app surfaces
  - allowed in chooser screens
- Release builds must use real AdMob IDs

## 6. Technical Structure

- Backend: Supabase database and Edge Functions
- Android app: Kotlin + Jetpack Compose
- Browser extension: Manifest V3
- Website: privacy, support, and terms pages

Key runtime capabilities:

- explicit device targeting
- Add Device pairing from Android Home
- rename/remove paired devices
- QR and 6-digit pairing
- browser identity reset
- minimal backend logs mode
- short-lived throttling and abuse protection
